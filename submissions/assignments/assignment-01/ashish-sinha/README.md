## 📖 Overview

This project demonstrates the implementation and comparison of multiple **Prompt Engineering techniques** using the **Groq API** and Large Language Models (LLMs).

The notebook explores how different prompting strategies influence model reasoning, decision-making, classification accuracy, response quality, and consistency across various business and technical scenarios.

The assignment covers real-world enterprise use cases such as:

- Vendor Risk Assessment
- Customer Support Ticket Classification
- API Contract Generation
- Business ROI Analysis
- Root Cause Investigation
- Security Incident Assessment
- AI Response Evaluation
- Architecture Decision Making
- Requirement Clarification

## 🎯 Learning Objectives

This project aims to:

- Understand core Prompt Engineering methodologies
- Compare different prompting approaches
- Improve LLM reasoning capabilities
- Evaluate AI-generated responses systematically
- Apply prompting techniques to real-world business scenarios
- Explore advanced reasoning frameworks used in production AI systems

# 🛠️ Prompt Engineering Techniques Implemented

## 1️⃣ Zero-Shot Prompting

The model is given only a task description without any examples.

### Case Study 1.1: Vendor Risk Classification

**Objective**
- Analyze vendor information
- Determine risk level
- Provide justification for classification

### Case Study 1.2: Executive Decision Memo Generation

**Objective**
- Generate structured executive summaries
- Support strategic business decisions
- Present concise recommendations

---

## 2️⃣ Few-Shot Prompting

The model receives example inputs and outputs before solving the target task.

### Case Study 2.1: Customer Support Ticket Classification

**Objective**
- Categorize customer support requests
- Learn classification patterns from examples

### Case Study 2.2: Requirement to API Contract Conversion

**Objective**
- Transform business requirements
- Generate structured API specifications

---

## 3️⃣ Chain-of-Thought (CoT) Prompting

Encourages the model to reason through a problem step-by-step before arriving at a final answer.

### Case Study 3.1: Business ROI Evaluation

**Objective**
- Analyze investment opportunities
- Evaluate costs and benefits
- Recommend the best option

### Case Study 3.2: ML Performance Root Cause Analysis

**Objective**
- Investigate performance degradation
- Identify potential causes
- Suggest corrective actions

---

## 4️⃣ LLM-as-Judge

Uses an LLM to evaluate and score outputs generated by another AI model.

### Case Study 4.1: Customer Support Response Review

**Evaluation Metrics**

- Accuracy
- Completeness
- Professionalism
- Helpfulness

### Case Study 4.2: Technical Explanation Assessment

**Evaluation Metrics**

- Technical Correctness
- Clarity
- Educational Value
- Depth of Explanation

---

## 5️⃣ Self-Consistency Prompting

Generates multiple reasoning paths and selects the most consistent answer.

### Case Study 5.1: Policy Interpretation

**Process**

- Execute multiple independent runs
- Aggregate responses
- Select consensus answer

### Case Study 5.2: Security Incident Risk Assessment

**Process**

- Evaluate security events
- Generate multiple risk assessments
- Apply majority voting

---

## 6️⃣ Tree-of-Thought (ToT)

Explores multiple reasoning branches before selecting the best solution.

### Case Study 6.1: AI Automation Opportunity Selection

**Objective**

Compare multiple AI use cases and determine the highest-value opportunity.

### Case Study 6.2: System Architecture Selection

**Objective**

Evaluate architectural alternatives and identify the optimal design.

---

## 7️⃣ Rephrase-and-Respond

Improves unclear user requests before generating responses.

### Case Study 7.1: Business Requirement Clarification

**Objective**

Transform ambiguous requests into actionable requirements.

### Case Study 7.2: Technical Requirement Enhancement

**Objective**

Improve poorly written technical specifications.

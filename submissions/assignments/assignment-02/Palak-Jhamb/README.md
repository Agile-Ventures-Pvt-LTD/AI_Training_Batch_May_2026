## 3. Short Description of Assignment

This project implements an **Advanced Retrieval-Augmented Generation (RAG)** pipeline for answering questions from Tesla annual reports (10-K filings).

The system enhances traditional RAG by using **Query Expansion**, where a user's question is automatically rewritten into multiple semantically related queries. These expanded queries retrieve more relevant document chunks from a vector database, improving the quality and completeness of the final answer.

### Key Features

* Loads Tesla annual report PDFs.
* Splits documents into manageable text chunks.
* Generates embeddings using **Sentence Transformers**.
* Stores embeddings in a **ChromaDB** vector database.
* Performs **Query Expansion** using an LLM (Groq GPT-OSS-120B).
* Retrieves relevant document chunks for all expanded queries.
* Combines retrieved context and generates an accurate answer using an LLM.
* Improves retrieval quality compared to standard single-query RAG systems.

### Workflow

1. Load Tesla 10-K reports.
2. Split reports into chunks.
3. Generate embeddings for chunks.
4. Store embeddings in ChromaDB.
5. Accept user question.
6. Generate multiple expanded versions of the question.
7. Retrieve relevant chunks for each expanded query.
8. Merge retrieved context.
9. Generate final answer using the retrieved information.

## 6. Explanation

### Query Expansion Example

**User Query**

```text
What was the automotive revenue in 2021?
```

**Expanded Queries Generated by LLM**

```text
What was Tesla's automotive revenue in 2021?
How much revenue did Tesla generate from automotive sales in 2021?
What were Tesla's automotive segment revenues during fiscal year 2021?
```

### Retrieval Phase

The retriever searches the vector database for each expanded query and collects the most relevant document chunks.

### Final Answer Generation

The retrieved chunks are merged and provided to the LLM as context.

Example Output:

```text
Tesla reported automotive revenue of approximately $47.2 billion in 2021, representing significant growth compared to the previous year.
```

### Benefits of Query Expansion

* Improves retrieval recall.
* Handles different phrasings of the same question.
* Retrieves more comprehensive context.
* Produces more accurate answers.
* Reduces chances of missing relevant information.
